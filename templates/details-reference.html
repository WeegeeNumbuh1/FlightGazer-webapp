What do all these logs mean?

  • Live Details - Live data being processed by FlightGazer.
    • For an explanation for all the data points, refer to the online guide.
    • To programmatically access this same data, check the /data/current_state.json endpoint.

  • Main Log - Logfile of FlightGazer events.
    • This file keeps a history of the times FlightGazer is running. It records startup configuration, issues with certain settings and features, and any errors that may arise.
    • After the “Main loop started!” line, FlightGazer tracks special events and generates stats at the end of each day.
    • If FlightGazer is running in a degraded state, this log will explain why.

    Some phrases you may encounter in the log
    • “High-priority aircraft override triggered by”...
      • Indicates that this aircraft was within 0.4 or less nautical miles line-of-sight from your site while there were also other aircraft in the area. This causes the aircraft selection algorithm to override its selection process and only track this High-priority aircraft until it is no longer within this distance.
    • “rare selections”
      • Number of times the amount of aircraft in the designated tracking area changed when the selection algorithm was about to pick another aircraft, forcing it to handle this “rare” case.
    • “aircraft selections”
      • Number of times the selection algorithm had to select an aircraft. The busier the site, the higher this number will be compared to flybys.
    • “Could not find database entry for”...
      • Indicates that this aircraft with this ICAO hex ID does not have an entry in the lookup database (e.g. there is no information about it).

  • Initialization Log - Startup information before the main FlightGazer program runs.
    • This is a system-level log that keeps track of startup tasks and any required preparation handled by the system before running FlightGazer.
    • If FlightGazer fails to start, it's important to check this log.
    • If its been awhile since FlightGazer was started, checking this tab may show “No entries”. Depending on the system, this log may have been rotated (think: archived) and thus entries for this log are no longer accessible.

  • Hourly Data - Hourly data of flybys and API usage.
    • This comma-separated values file keeps track of cumulative hourly flybys and API usage as FlightGazer runs.
    • This file is used by FlightGazer so that the flyby count and API usage isn't lost whenever it's restarted.
    • The data in this file can be used for long term traffic analysis of your site.
    • In a year, this file grows by roughly 250 KiB.

  • Migration History - Logfile of FlightGazer updates.
    • This file keeps track of every instance FlightGazer is updated. Specifically, this log records the settings migration portion of the update process.
    • This log is important to check if there are any issues when updating FlightGazer.
    • Future releases may add new features or change some settings, and this log compares what needed to be changed from the default settings to match your current settings at the time.
