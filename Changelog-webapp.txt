v.0.13.0 - 2025-11-09
    - Added logging
    - Add some hardening when dealing with the startup configuration to prevent command injection style attacks
    - Update link for Skystats
    - Updated Tips & Tricks section to mention new endpoint

v.0.12.3 - 2025-11-06
    - Added new endpoint '/data/current_state.json' to allow direct polling of the state file
    - When updating the FlightGazer service file, the directory is now enclosed in quotes

v.0.12.2 - 2025-10-31
    - Fix landing page never showing the "degraded state" status message

v.0.12.1 - 2025-10-30
    - When reading the flyby stats, the latest 1000 lines are read instead of the whole file
      for both client-side responsiveness and reducing the need to scroll through so many rows
    - Minor layout tweaks for all pages
    - Fixed minor UI glitch on the landing page when starting the service and it's already running

v.0.12.0 - 2025-10-29
    - Details page improvements:
        - Live Details tab no longer refreshes upon navigation to the tab from other tabs
        - There's now a time stamp on when the data was accessed
        - When downloading the state file it now reflects what is currently being displayed and not
          the state of the file itself when the download button is pressed
        - The keys in the Live Details page are no longer sorted alphabetically
        - When viewing the flybys csv, the header is now sticky
        - Improved the csv rendering from the backend
        - Further improved handling state file errors with better descriptions of the errors
        - More descriptive text for cases when the other logs cannot be read
    - Make API schedule table scrollable
    - Removed ability to see/configure the WRITE_STATE and FLYBY_STATS_ENABLED settings

v.0.11.2 - 2025-10-28
    - When downloading the initialization log, the date and time is appended as well

v.0.11.1 - 2025-10-28
    - Improved handling when the state file is not able to be read (ex: FlightGazer is not running)
    - When downloading logs, the date and time are appended at the end of the filename

v.0.11.0 - 2025-10-28

    - Additional tweaks to the Reference page
